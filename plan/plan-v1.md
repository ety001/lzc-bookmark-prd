## 1. 项目概述

### 1.1 产品目标

基于懒猫微服架构的 Chrome 浏览器书签同步系统，实现多设备间书签的自动备份与同步。

### 1.2 核心功能

- 跨设备同步：多个 Chrome 浏览器安装插件后，书签自动同步
- 版本历史：保留书签变更历史，支持回滚
- 私有部署：数据存储在用户自己的懒猫微服
- 增量同步：高效同步，只传输变更数据

### 1.3 技术栈

| 组件 | 技术栈 | 版本 |
|------|--------|------|
| 服务端 | Go + Gin + GORM | 1.23+ |
| 数据库 | SQLite | 3.35+ |
| Web 管理端 | React + Vite + shadcn/ui | v18+ |
| Chrome 插件 | Manifest V3 + Preact | 102+ |
| 部署 | Docker + 懒猫微服 | - |

### 1.4 关键设计决策

- **先推后拉**：客户端先推送本地变更，再拉取远程变更
- **双层 ID 映射**：服务端 GUID 规范树 + 插件本地 local_id 映射
- **变更日志**：每条变更版本号 +1，支持增量同步
- **冲突检测**：基于 base_version 与最新 to_version 比较
- **部分冲突**：无冲突变更正常应用，冲突变更挂起

## 10. 风险与应对

### 10.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| SQLite 版本兼容性 | 低于 3.35 不支持 RETURNING | 中 | Docker 镜像固定 SQLite 3.35+ |
| Chrome API 变更 | Manifest V3 API 不稳定 | 低 | 使用官方稳定 API，避免实验性 API |
| Service Worker 休眠 | 事件丢失导致同步延迟 | 中 | 实现唤醒时全量 diff 检测 |

### 10.2 进度风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 同步逻辑复杂 | 开发时间超出预期 | 中 | Phase 3 预留 1 天缓冲 |
| 多设备测试环境 | 测试环境搭建耗时 | 低 | 提前准备测试设备 |
| Web 管理端 UI | shadcn/ui 学习曲线 | 低 | 参考官方示例，简化定制 |

### 10.3 质量风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 并发冲突处理 | 边界情况未覆盖 | 中 | 编写详细测试用例，覆盖边界场景 |
| 数据一致性 | 同步过程中数据丢失 | 低 | 实现事务保证，编写恢复脚本 |
| 用户体验 | 冲突解决流程复杂 | 中 | 简化 UI 流程，提供明确引导 |
